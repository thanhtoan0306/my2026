# Connect another backend to this Firestore database

Guide for wiring a **new backend project** (Go, Node, Python, etc.) to the same Firebase Firestore data used by `todo-htmx` and `firebase-sticky-notes`.

---

## 1. What you are connecting to

| Item | Value |
|------|--------|
| **Product** | Firebase **Firestore** (not Realtime Database) |
| **Database** | `(default)` |
| **Project ID** | Your Firebase project, e.g. `gen-lang-client-0298822444` |
| **Region** | Whatever you chose when creating Firestore (e.g. `asia-southeast1`) |

### Collection paths in this repo

| App | Path | Document fields |
|-----|------|-----------------|
| **todo-htmx** (Go) | `artifacts/{appId}/public/data/todos/{todoId}` | `title`, `done`, `createdAt` |
| **firebase-sticky-notes** (browser) | `artifacts/{appId}/public/data/notes/{noteId}` | `title`, `content`, `authorId`, `createdAt` |

`{appId}` is usually the same as **project ID** (`FIREBASE_APP_ID` / `projectId` in config).  
Example full path for todos:

```text
artifacts/gen-lang-client-0298822444/public/data/todos/abc123xyz
```

## 2. Credentials: two ways to connect

### A) Server backend (recommended) — Service account

Used by **todo-htmx** today. Full read/write; **ignores** Firestore security rules.

1. Firebase Console → **Project settings** (gear) → **Service accounts**
2. **Generate new private key** → save JSON (e.g. `serviceAccount.json`)
3. **Never commit** this file (already in `.gitignore`)

```bash
export GOOGLE_APPLICATION_CREDENTIALS="/absolute/path/to/serviceAccount.json"
export FIREBASE_PROJECT_ID="your-project-id"
export FIREBASE_APP_ID="your-project-id"   # optional; defaults to project ID
```

### B) Client / mobile / SPA — Web config + Auth

Used by **firebase-sticky-notes**. Subject to **security rules**; users must sign in (e.g. Anonymous).

Copy from Firebase Console → Project settings → Your apps → Web app → **Config**:

```json
{
  "apiKey": "...",
  "authDomain": "your-project.firebaseapp.com",
  "projectId": "your-project-id",
  "storageBucket": "your-project.firebasestorage.app",
  "messagingSenderId": "...",
  "appId": "..."
}
```

Enable **Authentication → Anonymous** (or your provider) and publish rules (section 6).

## 3. Environment variables (server backends)

| Variable | Required | Description |
|----------|----------|-------------|
| `GOOGLE_APPLICATION_CREDENTIALS` | Yes (server) | Path to service account JSON |
| `FIREBASE_PROJECT_ID` | Yes* | GCP / Firebase project ID |
| `FIREBASE_APP_ID` | No | Path segment under `artifacts/`; default = project ID |

\*Or put `"projectId"` in `config.json` next to your app (see `todo-htmx/config.json.example`).

## 4. Document schema (todos)

Match what `todo-htmx` writes:

```json
{
  "title": "Buy milk",
  "done": false,
  "createdAt": "2026-05-26T10:00:00Z"
}
```

| Field | Type | Notes |
|-------|------|--------|
| `title` | string | Required for display |
| `done` | boolean | Toggle state |
| `createdAt` | timestamp | Use Firestore timestamp type; sort DESC in lists |

Document ID: auto-generated by Firestore (`NewDoc()` / `addDoc()`).

## 5. Code examples

### Go (same as todo-htmx)

```bash
go get cloud.google.com/go/firestore
```

```go
package main

import (
    "context"
    "log"
    "time"

    "cloud.google.com/go/firestore"
)

func todosCollection(ctx context.Context, projectID, appID string) (*firestore.CollectionRef, error) {
    client, err := firestore.NewClient(ctx, projectID)
    if err != nil {
        return nil, err
    }
    return client.Collection("artifacts").Doc(appID).
        Collection("public").Doc("data").Collection("todos"), nil
}

func main() {
    ctx := context.Background()
    projectID := "your-project-id"
    appID := "your-project-id"

    col, err := todosCollection(ctx, projectID, appID)
    if err != nil {
        log.Fatal(err)
    }

    // Create
    ref := col.NewDoc()
    _, err = ref.Set(ctx, map[string]interface{}{
        "title":     "From new backend",
        "done":      false,
        "createdAt": time.Now().UTC(),
    })
    if err != nil {
        log.Fatal(err)
    }
    log.Println("created", ref.ID)
}
```

Run:

```bash
export GOOGLE_APPLICATION_CREDENTIALS="$(pwd)/serviceAccount.json"
go run .
```

Reference implementation: [`todo-htmx/store_firestore.go`](todo-htmx/store_firestore.go).

### Node.js (Firebase Admin SDK)

```bash
npm install firebase-admin
```

```javascript
import { initializeApp, cert } from "firebase-admin/app";
import { getFirestore, Timestamp } from "firebase-admin/firestore";

initializeApp({
  credential: cert("./serviceAccount.json"),
});

const projectId = "your-project-id";
const appId = "your-project-id";

const db = getFirestore();
const todos = db
  .collection("artifacts")
  .doc(appId)
  .collection("public")
  .doc("data")
  .collection("todos");

// Create
const ref = todos.doc();
await ref.set({
  title: "From Node backend",
  done: false,
  createdAt: Timestamp.now(),
});

// List (newest first)
const snap = await todos.orderBy("createdAt", "desc").limit(50).get();
snap.forEach((doc) => console.log(doc.id, doc.data()));
```

```python
cred = credentials.Certificate("serviceAccount.json")

    db.collection("artifacts")
    .document(app_id)
    .collection("public")
    .document("data")
    .collection("todos")
)

todos.add({
    "title": "From Python backend",
    "done": False,
    "createdAt": firestore.SERVER_TIMESTAMP,
})
```

## 6. Firestore security rules (browser / client SDK only)

Server Admin SDK **bypasses** rules. If your new project uses the **client** SDK, publish rules like:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /artifacts/{appId}/public/data/todos/{todoId} {
      allow read, write: if request.auth != null;
    }
    match /artifacts/{appId}/public/data/notes/{noteId} {
      allow read, write: if request.auth != null;
    }
  }
}
```

Copy from: [`firebase-sticky-notes/firestore.rules`](firebase-sticky-notes/firestore.rules)  
Console: `https://console.firebase.google.com/project/YOUR_PROJECT_ID/firestore/rules` → **Publish**

## 7. Indexes

`todo-htmx` lists todos with:

```text
ORDER BY createdAt DESC
```

If Firestore asks for a composite index, open the link in the error message or create:

- Collection: `todos` (collection group or under `artifacts/.../data/todos`)
- Field: `createdAt` — Descending

## 8. View data in Firebase Console

1. [Firestore Data](https://console.firebase.google.com/project/YOUR_PROJECT_ID/firestore/databases/-default-/data)
2. Expand: **artifacts** → **{appId}** → **public** → **data** → **todos**
3. Click a document to see `title`, `done`, `createdAt`

Replace `YOUR_PROJECT_ID` with your real project ID.

---

## 9. Checklist for a new backend

- [ ] Same Firebase **project ID** as this repo
- [ ] Service account JSON + `GOOGLE_APPLICATION_CREDENTIALS` (server), **or** web config + Auth (client)
- [ ] Collection path: `artifacts/{appId}/public/data/todos` (or `notes` for sticky notes)
- [ ] Same field names and types as existing apps
- [ ] Rules published if using client SDK
- [ ] Create composite index if queries use `orderBy("createdAt")`
- [ ] Do not commit `serviceAccount.json` or `config.json` with secrets

## 11. Troubleshooting

| Error | Fix |
|-------|-----|
| `Could not load default credentials` | Set `GOOGLE_APPLICATION_CREDENTIALS` to service account path |
| `Permission denied` (client SDK) | Publish rules; sign in user (Anonymous, etc.) |
| `NOT_FOUND` on project | Wrong `FIREBASE_PROJECT_ID` |
| `The query requires an index` | Create index from Console link in error |
| Data not visible in Console | Wrong path; check `artifacts/{appId}/public/data/todos` |
| Go works but browser fails | Normal: server uses Admin SDK; browser needs rules + auth |

## 12. Sharing config between repos

Copy from this monorepo:

```bash
# Service account (keep secret)
cp todo-htmx/serviceAccount.json /path/to/new-backend/serviceAccount.json

# Project ID only
cp todo-htmx/config.json.example /path/to/new-backend/config.json
# edit projectId
```

Or reuse `firebase-sticky-notes/config.json` for `projectId` when both apps target the same Firebase project.
