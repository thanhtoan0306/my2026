Khởi tạo lần đầu

Cài Docker Desktop, clone repo, vào thư mục project, build + chạy lần đầu.

cd {{.ProjectDir}}

2

Chạy lần sau

Đã build image rồi — chỉ cần bật container (nhanh hơn, không cần --build trừ khi đổi code).

cd {{.ProjectDir}}
docker compose up -d
docker compose logs -f

Dừng: docker compose down

Chạy từ image trên Docker Hub (máy khác):

docker pull {{.DockerImage}}
docker run -d -p 8080:8080 --name ssr-hello {{.DockerImage}}
3

Đẩy lên Docker Hub

Đăng nhập Hub, tag image, push. Thay your-dockerhub-user bằng username của bạn.

cd {{.ProjectDir}}
docker login
docker build -t {{.DockerImage}} .
docker push {{.DockerImage}}

Sau khi push, người khác chỉ cần docker pull {{.DockerImage}} rồi docker run như bước 2.

4

Offline — bản Docker trong output/

Export image ra file {{.OfflineTar}}, copy cả folder project (hoặc chỉ output/ + scripts/) sang máy không có mạng.

Máy có mạng — tạo bản offline:

cd {{.ProjectDir}}
./scripts/export-offline.sh

Máy offline — load & chạy (không cần Hub / pull):

cd {{.ProjectDir}}
./scripts/run-offline.sh

Thủ công:

docker load -i {{.OfflineTar}}
docker run -d -p 8080:8080 --name ssr-hello-offline {{.OfflineImage}}

Image tag: {{.OfflineImage}}

!

Case: Docker daemon chưa chạy

Gặp lỗi khi docker compose up — Docker Desktop tắt hoặc chưa khởi động xong:

Cannot connect to the Docker daemon at unix:///Users/fe.tony/.docker/run/docker.sock. Is the docker daemon running?

Cách xử lý (macOS):

# 1. Mở Docker Desktop, đợi icon cá voi sáng / "Docker Desktop is running"
open -a Docker

# 2. Kiểm tra daemon đã sẵn sàng
docker info

# 3. Chạy lại app
cd {{.ProjectDir}}
docker compose up -d

Hoặc gộp một lệnh:

open -a Docker && sleep 5 && cd {{.ProjectDir}} && docker compose up -d

Nếu vẫn lỗi: mở app Docker Desktop thủ công từ Applications, đợi ~30s rồi thử lại.

Gợi ý: Đặt tên image qua biến môi trường DOCKER_IMAGE=tenban/docker-ssr-hello:latest khi chạy app hoặc trong file .env.
